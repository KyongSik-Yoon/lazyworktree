## Project

Go bubbletea TUI for managing GIT worktrees. Read the README if you really need
to know what this project do.

## Building

- use `make build` for testing build errors.

## Before Finishing

- Always Run `make sanity` which will run golangci-lint, gofumpt, and go test.
- Consider adding tests for any new functionality.
- Consider adding README.md updates for any user-facing changes. Do not sound like
it is written by a braindead overenthusiastic robot.
